# v2.0.0 - 运行时简拼算法版本

**发布日期：** 2026-07-08

## 🎯 核心改进

### 简拼匹配改为运行时算法

**问题：** 之前版本预生成简拼编码到词库，导致编码冲突
- `zaohong` (枣红) 生成了 `zh` 简拼编码
- 输入 `zhong` 会错误匹配到 "枣红"

**解决方案：** 改用运行时动态简拼匹配
- 词库只存储完整拼音编码
- 简拼在运行时动态提取和匹配
- 彻底避免预生成编码冲突

## 📊 词库优化

- **大小优化：** 127,025 → 55,219 条目（减少 57%）
- **编码清理：** 删除所有预生成的简拼编码
- **冲突解决：** 消除 zh/ch/sh 等声母冲突

## ✅ 匹配优先级

1. **完整匹配** - 精确匹配（编码长度 == 输入长度）
2. **运行时简拼** - 动态提取声母匹配
3. **逐字匹配** - 最后的 fallback

## 🐛 问题修复

| 问题 | 状态 | 说明 |
|------|------|------|
| zhong 匹配到枣红 | ✅ 已修复 | zaohong 不再有 zh 简拼编码 |
| zh 显示简拼词组 | ✅ 已修复 | 跳过简拼，只显示单字 |
| 简拼冲突 | ✅ 已解决 | 改用运行时算法 |

## ✅ 测试验证

- ✅ 输入 `zhong` → 显示 "中、重、钟"，不显示 "枣红"
- ✅ 输入 `zh` → 显示 zh 开头单字（中、只），不显示 "最后"
- ✅ 输入 `zg` → 匹配 "中国"（运行时简拼）
- ✅ 输入 `zhg` → 匹配 "中国"（运行时简拼）

### 刷写方法

#### 使用 esptool
```bash
esptool --chip auto --port /dev/ttyUSB0 --baud 921600 \
  write_flash -z 0x0 typewriter-pinyin-v2.0.0-runtime-sh.bin
```

#### 使用 M5Burner
1. 下载 M5Burner: https://docs.m5stack.com/en/download
2. 选择固件文件
3. 选择设备端口
4. 点击 "Burn" 刷写

## 🔧 技术细节

### 词库生成改进
- 删除 `gen_ime.py` 中的简拼预生成逻辑
- 只 yield 完整拼音编码到词库
- 词库大小减少 57%

### 匹配逻辑优化
- 词组精确匹配：`(int)cl == qlen && strncmp(wc, q, qlen) == 0`
- 运行时简拼提取：动态提取每个音节的声母
- zh/ch/sh 特殊处理：跳过简拼，优先显示单字
